XFetch Documentation

Universal HTTP fetching library — framework-agnostic, SSR-safe, TypeScript-first, zero runtime dependencies.

Features

  • Zero dependencies: Only uses native fetch. Compatible with Node 18+, browsers, Edge Runtime, Bun, and Deno.
  • Smart Retry: Built-in exponential backoff with jitter and customizable status codes.
  • Caching: Memory and LocalStorage backends, TTL-based eviction.
  • Interceptors: Axios-style request, response, and error middleware chains.
  • Security hardened: SSRF protection, prototype-pollution defenses, and safe headers handling.
  • Framework Ready: Includes adapters for React, Vue, Next.js, and Nuxt.

Installation

Install the library from npm using your preferred package manager:

npm install @ar-coder/xfetch
# or
pnpm add @ar-coder/xfetch
# or
yarn add @ar-coder/xfetch

For usage without a bundler, you can also load it via CDN:

<script src="https://cdn.jsdelivr.net/npm/@ar-coder/xfetch/dist/xfetch.umd.js"></script>

Basic Usage

Creating an instance encapsulates your base URL, default headers, and global behavior.

import { createClient } from "@ar-coder/xfetch";

const api = createClient({
  baseURL: "https://api.example.com",
  headers: {
    Accept: "application/json",
  },
  timeout: 10000,
});

GET & POST Methods

Standard HTTP methods are exposed directly on the client instance.

// GET with automatic TypeScript inference
const { data: users } = await api.get<User[]>("/users");

// POST — objects are automatically serialized to JSON
await api.post("/users", { name: "Anwar", role: "admin" });

// Seamless REST support
await api.put("/users/1", { name: "Updated" });
await api.delete("/users/1");

API: createClient

Accepts a configuration object to setup a new HTTP client instance.

const api = createClient({
  baseURL?: string;       // Base URL for all requests
  headers?: Record<string, string>; // Global headers
  timeout?: number;       // Default timeout in ms
  retry?: RetryOptions;   // Global retry rules
  cache?: CacheOptions;   // Global cache rules
  auth?: AuthOptions;     // Configuration for Bearer tokens
  debug?: boolean;        // Enables safe request/response logging
});

API: Interceptors

Modify requests or responses before they reach your app logic. Similar to Axios.

// Request Interceptor
api.interceptors.request.use(async (ctx) => {
  ctx.headers["Accept-Language"] = "en-US";
  return ctx;
});

// Response Interceptor
api.interceptors.response.use((res) => {
  console.log('Got response', res.status);
  return res;
});

// Error Interceptor
api.interceptors.error.use((error) => {
  if (error.status === 403) window.location.href = "/login";
  return Promise.reject(error);
});

API: Caching

Stop waiting on redundant data. Supports memory and cross-tab LocalStorage.

// Memory cache, lives for 5 minutes
await api.get("/features", { cache: { storage: 'memory', ttl: 300000 } });

// LocalStorage cache for persistent state
await api.get("/config", { cache: { storage: 'localStorage' } });

// Explicitly bypass cache
await api.get("/status", { cache: false });

API: Retry Logic

Automatically retries idempotent requests (GET, PUT, etc.) using exponential backoff.

const api = createClient({
  retry: {
    count: 3, 
    delay: 500, // wait 500ms -> 1000ms -> 2000ms
    statusCodes: [408, 429, 500, 502, 503, 504],
  }
});

API: Authentication

Built-in 401 Unauthorized refresh flow. No custom interceptor required to keep tokens fresh.

const api = createClient({
  auth: {
    token: "initial-token",
    // Automatically pauses queue and refreshes on 401
    refreshToken: async () => {
      const res = await fetch("/api/refresh", { method: 'POST' });
      const json = await res.json();
      return json.newToken;
    }
  }
});

Examples: React

Extract reactive states directly in your React components.

import { useRequest } from "@ar-coder/xfetch/react";

function UserList() {
  const { data, loading, error } = useRequest<User[]>(api, "/users");

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error!</div>;

  return <ul>{data?.map(u => <li key={u.id}>{u.name}</li>)}</ul>;
}

Examples: Vue

Native integration with Vue's Reactivity system.

<script setup>
import { useApi } from "@ar-coder/xfetch/vue";

const { data, loading } = useApi(api, "/users");
</script>
<template>
  <div v-if="loading">Loading...</div>
  <ul v-else>
    <li v-for="u in data" :key="u.id">{{ u.name }}</li>
  </ul>
</template>

Examples: Next.js (App Router)

Runs perfectly on the server without leaks.

export default async function Page() {
  // SSR deduping works out of the box
  const { data: users } = await api.get<User[]>("/users");
  return <UserList users={users} />;
}

Examples: Vanilla JS & jQuery

XFetch works seamlessly in the browser via CDN script tags.

<!-- Load XFetch from CDN -->
<script src="https://cdn.jsdelivr.net/npm/@ar-coder/xfetch/dist/xfetch.umd.js"></script>
<script>
  // It attaches to the global window object
  const api = window.XFetch.createClient({ baseURL: 'https://api.example.com' });

  // 1. Vanilla JS Usage
  document.getElementById('load-btn').addEventListener('click', async () => {
    try {
      const res = await api.get('/users');
      console.log('Vanilla JS Data:', res.data);
    } catch (error) {
      console.error('Error fetching users:', error);
    }
  });

  // 2. jQuery Usage (Combining the power of XFetch with DOM manipulation)
  $('#jquery-btn').on('click', function() {
    api.get('/users').then(res => {
      res.data.forEach(user => {
        $('#user-list').append(`<li>${user.name}</li>`);
      });
    });
  });
</script>
